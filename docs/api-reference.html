日報管理ダッシュボード API (1.0.0)

Download OpenAPI specification:

License: MIT

Markdown形式で管理している日報データをグラフィカルに表示・編集するための Webダッシュボードアプリケーション用のREST API仕様です。

設計方針

  • 日報データはMarkdownファイルとして管理し、フロントエンドでパース・編集
  • サーバーサイドはファイルの読み書きと、複数日にまたがる集計処理を担当
  • 外部連携（Git commit/push、Slack投稿）は日報保存時のオプションとして実行
  • 連携設定（Slack/Git/GCal）はサーバーサイドで管理、APIには露出しない

日報ファイル形式

日報ファイルはYAML frontmatter + Markdown本文で構成されます。 frontmatterには統計情報が含まれ、カレンダー表示時の高速な集計を可能にします。

---
planHours: 8.0
resultHours: 7.5
todoCount: 5
todoCompleted: 2
updatedAt: 2025-12-18T17:30:00+09:00
---
# [日報] サンプル太郎 2025-12-18

## [PLAN]
* 08:30 [P99] タスク確認
...

Reports

日報ファイルの操作（Markdownベース）

日報取得

指定した日付の日報を取得します。 レスポンスにはMarkdown本文と、frontmatterから抽出した統計情報が含まれます。 日報が存在しない場合は404を返します。

date
required
string <date>
Example: 2025-12-18

日付（YYYY-MM-DD形式）

Responses

ローカル開発サーバー

http://localhost:3000/api/reports/{date}

日報保存

指定した日付の日報を保存します。 日報が存在しない場合は新規作成、存在する場合は上書き更新します。

サーバーサイド処理:

  • Markdownをパースして統計情報（計画時間、実績時間、TODO数等）を計算
  • 統計情報をYAML frontmatterとしてファイル先頭に追加して保存
  • これにより、カレンダー表示時に高速な集計が可能

オプションでGit commit/push、Slack投稿を同時に実行できます。

path Parameters
date
required
string <date>
Example: 2025-12-18

日付（YYYY-MM-DD形式）

Request Body schema: application/json
required
content
required
string

日報のMarkdownコンテンツ（frontmatterなしの本文のみ）。 サーバーが統計情報を計算してfrontmatterを付与して保存します。

object

Git操作オプション

object

Slack投稿オプション

Responses

http://localhost:3000/api/reports/{date}

Calendar

カレンダー用の集計データ

カレンダー用集計データ取得

指定した年月の日別統計データを取得します。

高速化: 各日報ファイルのfrontmatterから統計情報を読み取るため、 Markdown本文のパースは不要で高速に集計できます。

required
integer [ 2000 .. 2100 ]
Example: year=2025

年（YYYY形式）

month
required

月（1-12）
